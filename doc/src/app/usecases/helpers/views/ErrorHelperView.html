ErrorHelper

This Service assists with the displaying of error messages. At the core, it uses SweetAlert2 (more details here), which is a required dependency.

The error parsing code of the class has a lot of checks for thing such as GraphQL errors, validation errors and others already built in. Should these not fit your implementation, see the section below called Overriding all error parsing

Known vs. Unknown Errors

There's two APIs available. One is called displayError(...) and the other displayKnownError(...). The difference is that you are saying "I know what error I want to display", whereas the other is more intended for a try/catch scenario where you do not know what the error may be.

Overriding all error parsing

If you provide an implementation of the interface IErrorOverrideHandler and inject it with injection token ERROR_OVERRIDE_HANDLER, you can override all error handling.
